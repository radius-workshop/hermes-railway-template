Agent-to-Agent
🤝

A2A task delegation

Any A2A-compatible agent can discover this one, verify its cryptographic identity, and delegate tasks over HTTP — no pre-shared secrets needed.

💸

Agent-to-agent payments

Agents settle work on-chain using their built-in Radius wallets. The same key signs JWTs and transactions — identity and value travel together.

🔐

DID-based auth

Every agent has a did:web identity resolvable over HTTPS. JWT-gated endpoints verify callers cryptographically, not with API keys.

How it works

1

Deploy from the Railway template

Click Deploy. Railway spins up a containerized worker service backed by a persistent volume at /data.

2

Set your environment variables

Add your messaging token, LLM API key, and any optional extras — wallet key, Linear token. That's the entire config surface.

3

First boot bootstraps everything

Entrypoint validates config, initialises the on-chain wallet, registers skills, and starts the agent — automatically.

4

Talk to your agent

Message it on Telegram, Discord, or Slack. Subsequent deploys reuse persisted state — no re-initialization needed.

5

Connect agents together

Set A2A_PEER_URL and TRUSTED_DIDS to wire two agents. They can delegate tasks and settle payments on-chain using their built-in wallets.
